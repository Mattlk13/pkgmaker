Knitr Extensions

Description

knit_ex is a utility function for running small knitr examples, e.g., to illustrate functionnalities or issues.

hook_try simply defines a function try in envir that prints the error message if any, and is called instead of base try.

Usage

knit_ex(x, ..., quiet = TRUE)

hook_try(before, options, envir)

Arguments

x
text to knit as a character vector
...
arguments passed to knit2html or knit
quiet
logical that indicates if knitting should be quiet (no progress bars etc..).
before
logical that indicates when the hook is being called: before or after the chunk is processed.
options
list of current knitr chunk options
envir
environment where the chunk is evaluated

Examples

,,library(knitr),knit_ex("1 + 1"),library(knitr),,# standard error message is caught,knit_ex("stop('ah ah')"),,# with try the error is output on stderr but not caughted by knitr,knit_ex("try( stop('ah ah') )"),,# no message caught,knit_ex(",```{r, include = FALSE},knit_hooks$set(try = pkgmaker::hook_try),```,,```{r, try=TRUE},try( stop('ah ah') ),```"),,,,
